# Serverless Video Compositing

Minimal Amplify Gen 2 demo that maps a video onto a quadrilateral on a background image.

## High-level architecture
Flow
- User uploads input video + background image to S3 via Amplify Storage.
- UI creates a RenderJob record (status=PENDING) in AppSync/DynamoDB.
- UI calls `startRender` mutation (AppSync → Lambda).
- `startRender` invokes `renderVideo` Lambda asynchronously.
- `renderVideo` downloads assets from S3, runs ffmpeg perspective warp, uploads output to S3, updates job status.
- UI polls job status and fetches a signed download URL when completed.

Services
- Storage: S3 (Amplify Storage)
- API/Data: AppSync (Amplify Data) backed by DynamoDB
- Compute: Lambda (`startRender`, `renderVideo`)

## Requirements
- Node 20/22 LTS (Gen 2 CLI is flaky on Node 23).
- pnpm 9+.
- AWS credentials (via `aws configure` or `AWS_PROFILE=yourprofile`).

## Setup (pnpm + Gen 2)
1. Configure AWS credentials:
   - `aws configure` (or set `AWS_PROFILE=yourprofile`)
   - or `pnpm --package=@aws-amplify/backend-cli@latest dlx ampx configure profile`
2. `pnpm install`
3. Start the Gen 2 sandbox (from repo root):
   - `pnpm --package=@aws-amplify/backend-cli@latest dlx ampx sandbox --profile default`
4. Wait for `amplify_outputs.json` to be generated in the project root.
5. Attach an ffmpeg Lambda Layer to `renderVideo` (see `amplify/functions/renderVideo/resource.ts`).

## Run
- `pnpm dev`

## Deploy
- `pnpm --package=@aws-amplify/backend-cli@latest dlx ampx deploy`
- `pnpm build`

## Key decisions & trade-offs
- Polling for job status instead of subscriptions for simpler UX and fewer moving parts.
- Lambda + ffmpeg for rendering to keep the system minimal; not optimized for large/long videos.
- Input video duration capped at ~10s to keep Lambda runtime predictable.
- Corners stored as JSON and parsed in the render Lambda for flexibility.
- Preview is directionally accurate (canvas) rather than pixel-perfect with the final ffmpeg render.

## Notes
- `amplify_outputs.json` is generated by `sandbox`/`deploy` and is required for the app to connect.
- Inputs live under `inputs/` in Storage; outputs under `outputs/`.
- The render Lambda expects ffmpeg binaries at `/opt/bin/ffmpeg` and `/opt/bin/ffprobe`.
